SEKS 知识检索台

给造船/工程资料做入库、检索和问答的轻量页面。上半区先查、先问；下半区再做资料入库，适合日常快速使用。

问答（Ask）

适合直接提问题。系统会先检索相关片段，再给出整理后的答案；如果配置了 LLM，会尽量生成更自然的最终回答。

快捷键：Ctrl / ⌘ + Enter 可直接提交。
LLM 高级设置（可选）
这里可以临时控制本次 Ask 是否启用 LLM、使用哪个模型、温度多少。API 地址和 Key 仍然走服务端配置，不在页面暴露。
结果展示在右侧“结果区”。

检索（Search）

适合先看原文证据。检索结果会直接展示命中的片段，便于你判断资料有没有入对、搜对。

建议先用 Search 看原文，再决定是否用 Ask 让系统整理答案。结果展示在右侧“结果区”。

结果区

尚无结果
先在左侧提交 Ask 或 Search。

使用说明

        

          
下面这些点最容易踩坑，先看一遍能省不少时间。

          
    
            
  • 页面默认与当前 API 同源调用，通常直接打开 http://127.0.0.1:18080/ 就能用。
    • 
            
  • 如果 API 在别处运行，可在顶部填写“API 地址”，例如 http://127.0.0.1:18080 或 Tailscale 域名。
    • 
            
  • 如果你是直接双击 HTML 用 file:// 打开，浏览器通常会因为 CORS 拒绝请求；建议始终从 API 页面入口打开。
    • 
            
  • 当前支持入库的文件类型主要是 .txt、.md、.pdf。
    • 
            
  • 如果改了 embedding 模型，旧文档建议重新 ingest，避免向量空间混用。
    • 
            
  • 如果给 /ask 配好了 LLM，回答模式会从 retrieval-only 提升为 LLM RAG；若 LLM 失效，会自动退回检索整理模式。
    • 
          

          
前端文件：app/static/index.html
